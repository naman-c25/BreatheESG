# Breathe ESG — Ingest & Review Console

A Django + React prototype that ingests emissions data from three real-world source shapes (SAP fuel/procurement, utility electricity, corporate travel), normalizes it, and gives an analyst a dashboard to review, edit, approve, and lock rows for audit.

This README is just the runbook. The reasoning lives in:

- **[MODEL.md](MODEL.md)** — the data model and why.
- **[DECISIONS.md](DECISIONS.md)** — every ambiguity I resolved and why.
- **[TRADEOFFS.md](TRADEOFFS.md)** — three things I deliberately did not build.
- **[SOURCES.md](SOURCES.md)** — research per source.

Read MODEL.md first.

## Run locally

Requirements: Python 3.11+, Node 18+.

### Backend

```bash
cd backend
python -m venv .venv
# Windows:  .venv\Scripts\activate
# *nix:     source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed
python manage.py generate_sample_pdf
python manage.py runserver 0.0.0.0:8000
```

The `seed` command creates:
- Two tenants (Acme Manufacturing Inc — US, and Globex DE GmbH — DE).
- An analyst user per tenant.
- Two facilities, the GHG category taxonomy, ~10 emission factors (DEFRA / EPA / IEA), and three configured sources.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173. Pick a tenant from the top bar. Upload one of:

- `backend/sample_data/sap_fuel_2025_04.csv`
- `backend/sample_data/utility_bill_2025_04.pdf` (generated by `generate_sample_pdf`)
- `backend/sample_data/travel_concur_2025_04.json`

Rows arrive in the review table. Click a row to open the detail pane with the raw payload, flags, audit trail, and Approve / Lock buttons.

## Endpoints (for poking around)

All API endpoints require an `X-Tenant-Id` header (the frontend sets it from the tenant picker).

```
GET    /api/tenants/                  unscoped — for the picker
GET    /api/sources/
GET    /api/facilities/
GET    /api/categories/
GET    /api/runs/
POST   /api/ingest/                   multipart: source=<id>, file=<binary>
GET    /api/activities/               filters: status, scope, source, facility, run, q
PATCH  /api/activities/<id>/
POST   /api/activities/<id>/approve/  body: { reason }
POST   /api/activities/<id>/lock/     body: { reason }
POST   /api/activities/bulk_approve/  body: { ids: [...] }
POST   /api/flags/<id>/dismiss/       body: { reason }   (reason required)
GET    /api/audit/?entity_id=<id>
GET    /api/dashboard/summary/
```

## Deploy (Render)

See [render.yaml](render.yaml). One web service for Django (with whitenoise serving the built React bundle), one Postgres instance. Steps:

1. `cd frontend && npm install && npm run build` (CI step in the Render build command).
2. Copy `frontend/dist/` to `backend/staticfiles/frontend/` (the build script does this).
3. `python manage.py collectstatic --noinput && python manage.py migrate && python manage.py seed && python manage.py generate_sample_pdf`.
4. `gunicorn esg.wsgi --bind 0.0.0.0:$PORT`.

Set env vars on Render:
- `DJANGO_SECRET_KEY` — generated.
- `DATABASE_URL` — supplied by the Postgres add-on.
- `DJANGO_DEBUG=0`.
- `DJANGO_ALLOWED_HOSTS=<your-render-domain>`.

## Known omissions

A short pointer to the long-form discussion in TRADEOFFS.md:

- No authentication (tenant via header).
- Synchronous ingestion (no Celery).
- Original PDFs/CSVs are parsed but not retained as files.
- No automated tests yet.
- No pro-rata bill splitting UI (schema supports it).
